Since all layout components and sections are complete, we will now transform the site into a high-performance, cinematic "Full-Page Scroll" experience using **GSAP** (GreenSock) and its **Observer** plugin.

### Phase 1: Installation & Setup
1. Install GSAP and its React bindings: `npm install gsap @gsap/react`
2. Create a global wrapper component (e.g., `MasterScrollController.tsx`) that will replace the standard `<main>` layout in `page.tsx`.

### Phase 2: Architectural Restructure (The CSS Core)
To make full-page transitions work perfectly, we must convert the page from a normal scrolling document into a fixed viewport:
1. Wrap the entire app in a `100vh`, `100vw`, `overflow-hidden` container.
2. Ensure every Section component is wrapped in an `absolute inset-0` container. This completely un-stacks them from the normal DOM flow and places them squarely on top of each other.
3. Automatically set the `z-index` so the active section is always on top, and initially hide all sections (`opacity: 0`, `visibility: hidden`) except Section 1.

### Phase 3: The GSAP Observer Logic (The Engine)
We will utilize GSAP's `Observer.create()` to monitor user intent without relying on actual CSS scrolling:
1. **Track State**: Maintain a mutable reference variable `currentIndex = 0` (pointing to sections 0 through 8).
2. **Animation Lock**: Use a boolean flag `isAnimating = false`. If a transition is currently playing, ignore any new mouse wheels or swipes. This is crucial to prevent chaotic, rapid-fire scrolling through all 9 sections in one flick.
3. **Event Detection**: Listen for `onUp`, `onDown`, cursor drag, and touch swipes to increment or decrement the `currentIndex`.

### Phase 4: Mapping the Cinematic Transitions
Instead of boring slides, we will map specific multi-step Timelines (`gsap.timeline()`) for moving between index states. 
*   **Intro ➔ About (0 -> 1)**: Scale down Section 1 (`scale: 0.8`), drop its opacity, and slice/wipe Section 2 into view from the bottom to honor the paper-cut transition.
*   **About ➔ News (1 -> 2)**: Smooth alpha-fade with a slight Y-axis lift (`y: 50`).
*   **News ➔ Official Video (2 -> 3)**: Deep Z-axis push. The News section scales up aggressively (`scale: 1.5`, `blur: 10px`) and fades out, revealing the cinematic video player underneath.
*   **Video ➔ Timeline (3 -> 4)**: Drag transition. Section 4 exits to the left (`xPercent: -100`), while Section 5 sweeps in from the right (`xPercent: 100` to `0`).
*   **Timeline ➔ Registration (4 -> 5)**: Standard elegant fade with staggered elements.
*   **Registration ➔ Journey (5 -> 6)**: The Registration CTA cards sink down (`y: 100`, opacity: 0), while the Journey 3D gallery emerges from a tight zoom to full size.
*   **Journey ➔ Previous Videos (6 -> 7)**: Reverse drag (enters from the top, exits to bottom).
*   **Previous Videos ➔ Excellence Awards (7 -> 8)**: A dramatic glow fade. The screen drops to pitch black, and the Section 9 gold/red lights ignite over 1.5 seconds.

### Phase 5: Protecting Internal Scrolling (Crucial Step)
Several sections have internal horizontal scroll/swipe logic that must NOT trigger the global GSAP Observer:
*   **Section 3**: Newsroom carousel.
*   **Section 5**: 8-node Zig-Zag hover triggers (ensure hovers don't register as swipe intents).
*   **Section 7**: The 3D Curved Gallery (`overflow-x-auto snap-x`).
*   **Section 9**: The Horizontal Category Cards (`overflow-x-auto snap-x`).

**Solution**: GSAP Observer will be configured to `ignore: ".hide-scrollbars, .internal-scroll"`. We will add an `.internal-scroll` class to those specific horizontal containers, guaranteeing the user can scroll those components left and right freely without accidentally leaping to the next section!
